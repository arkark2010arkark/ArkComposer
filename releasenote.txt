ArkComposer Release Notes

Version 1.1.3.7
Date: 2026-04-11

Summary
- Added a categorized SoundFont preset browser with search functionality for faster instrument selection.
- Added per-track SoundFont assignment so each track can use a different SoundFont file.
- Improved playback reload logic so note edits are properly reflected when pressing Play.
- Allowed clicking any measure to seek playback position regardless of editor mode while playing.
- Improved harmony inference and accompaniment generation with automatic reference track switching and dissonance post-correction.
- Added Measure Cleanup command that trims trailing empty measures and clears output filters from empty measures.
- Instrument preview now uses a dedicated MIDI channel so different instruments sound correctly in the preset browser.

SoundFont / Instrument
- Each track can now be assigned a different SoundFont file; playback loads and applies per-track SoundFont settings independently.
- Replaced the flat preset list with a two-panel browser: category list on the left and filtered presets on the right.
- Added ten instrument categories (Keyboard, Guitar/Bass, Strings, Brass, Woodwind, Synth, Percussion/Drums, Ethnic, SFX, Other) with localized labels.
- Presets are classified by GM program number for bank 0 and by name keywords for other banks.
- Added a real-time search bar that filters presets by name across all categories.
- Increased default browser size to 900x600 with resizable limits up to 1600x1200.
- Instrument preview now plays on a dedicated MIDI channel (ch 15) so the selected preset is heard correctly instead of always playing the first track's instrument.

Playback
- Playback now detects note-level edits and forces a full song reload when content has changed, instead of reusing a stale cached snapshot.
- Clicking a measure during playback now seeks to that position in any editor mode (Measure Select, Note Select, or Edit), not only in Measure Select mode.

Harmony / Accompaniment
- Improved harmony inference algorithm for more accurate chord estimation.
- Improved melody reference track detection for accompaniment generation with automatic track switching during generation.
- Added final dissonance post-correction pass during AI accompaniment to reduce clashing notes.

Editing
- Measure Cleanup now finds the last measure with notes across all tracks (previously only checked the melody track) and trims trailing empty measures accordingly.
- Measure Cleanup now clears output filter automation from measures that contain no notes.

Version 1.1.2.4
Date: 2026-04-08

Summary
- Improved responsiveness when opening large score files, importing long MIDI files, and switching between long-document tabs.
- Reduced long-score paint stalls by stopping unnecessary full-song chord-symbol recomputation during normal UI drawing.
- Added visible progress overlays for save and export operations so long PDF/audio/image/MIDI writes no longer look like the program has frozen.

Loading / Import / Layout
- Opening score files and importing MIDI now show an immediate loading overlay instead of leaving the UI unresponsive with no feedback.
- MIDI import now avoids over-expanding measure width from a small number of extremely dense measures, reducing oversized initial layout on imported songs.
- Changing measure width now refreshes layout size and scroll bounds together, preventing large blank scroll space after the score becomes shorter.

Tab Switching / UI Performance
- Tab switching no longer forces unnecessary playback-song rebuilds when the player is idle.
- Removed repeated AI-model discovery and unnecessary accompaniment-panel refresh work from tab activation.
- Eliminated extra palette relayout work that had no user-visible benefit during document switches.

Score Rendering Performance
- Reworked top chord-symbol caching so normal score paint only computes the visible/front needed range first instead of rebuilding the whole song.
- PDF page rendering now benefits from progressive chord-symbol cache expansion across pages instead of restarting full-score work on each draw.
- Strengthened chord-symbol cache invalidation so edits only discard and rebuild the affected suffix rather than forcing broad recomputation.

Save / Export Feedback
- `Save`, `Save As`, score export (`PDF/PNG/JPG`), and audio/MIDI export (`WAV/MP3/MIDI`) now run file writing on a worker thread.
- The score view keeps a `Saving...` overlay visible until the operation completes or fails, then shows the completion/failure dialog.
- Added protection against overlapping save/export requests so only one persistence task runs at a time.

Version 1.1.2.3
Date: 2026-04-08

Summary
- Improved score rendering performance for long and dense songs, especially during playback and scrolling.
- Reduced UI stalls caused by repeated full-score recomputation in the paint path.
- Refined playback follow behavior to avoid horizontal jump glitches when moving to the next staff line.

Rendering / Performance
- Added stronger clip-aware culling so score drawing skips more work outside the visible region.
- Reduced duplicate repaint work during edit-mode mouse movement and insert-preview updates.
- Reworked playback beat-to-measure lookup to use cached mapping instead of repeated linear scans.
- Added caching for chord-symbol guide computation so long scores no longer rebuild earlier harmony context on every paint.
- Added cached note-layer rendering for visible systems so playback updates can reuse prepared note graphics instead of rebuilding note spacing, beam, and tie rendering every frame.

Playback / Scrolling
- Improved playback auto-follow so the score starts following earlier and more consistently in long songs.
- Fixed a playback-follow side effect where the viewport could jump horizontally at the end of a visible staff line before returning to the next line.
- Reduced cases where playback, stop response, mouse wheel input, and score scrolling could feel delayed on very large scores.

Version 1.1.1.3
Date: 2026-04-06
Summary
- Turned `Edit Measure (MIDI)` into a modeless editor that can stay open while working in the main score.
- Fixed major stability problems around editing and preview playback in the measure editor.
- Improved playback cursor behavior and reduced unnecessary redraw work for smoother editing and playback feedback.

Edit Measure / Workflow
- `Edit Measure (MIDI)` now stays available alongside the main score, Undo/Redo, and other editing commands.
- The main score selection and the current measure shown in `Edit Measure (MIDI)` now stay in sync more reliably.
- Preview edits continue to use the shared song data and shared undo pipeline instead of acting like a separate working copy.

Playback / Stability
- Fixed crashes that could occur when editing notes while playback or preview playback was active.
- Fixed preview playback cases that could stop immediately, enter a long loading state, or behave inconsistently after moving to a different measure.
- Preview playback in `Edit Measure (MIDI)` is now focused on the current measure being edited.

Playback Display / Performance
- Improved the playback cursor in `Edit Measure (MIDI)` so it moves more smoothly and starts from the correct position more consistently.
- Reduced unnecessary redraw work for score playback updates by limiting repaint work to smaller changed areas.
- The main score no longer redraws its playback line during measure-editor preview playback.

Version 1.0.1.1
Date: 2026-03-21

Summary
- Reworked note spacing, tied-segment handling, and dense-measure readability across score rendering.
- Normalized unnecessary split/tied note representations on JSON/MIDI load and reduced differences between loaded data and pasted data.
- Improved accompaniment generation and top chord display by switching harmony inference from a single-track reference to a merged harmony reference.
- Cleaned up major GUI workflows including the left palette, player controls, and edit/selection modes.

Notation / Layout
- Unified displayed note X positions and hit-test X positions under the same spacing logic.
- Improved adaptive spacing using measure density, note duration, and visible symbol count including rests.
- In sparse measures, spacing now follows duration proportion more strongly while still guaranteeing a minimum width for short notes such as 32nd notes.
- Reduced cases where tied fragments consumed excessive horizontal space by rebuilding spacing from untied temporary notation.
- Separated tie rendering responsibilities from spacing responsibilities so tie graphics no longer inflate spacing by themselves.
- Relaxed automatic notehead shrinking so it is used only when local gaps are actually too tight.
- Added green insertion guide lines in edit mode for valid note start positions inside rest windows.

Import / Normalize
- Added canonical notation duration cleanup for JSON/MIDI import when values are close to standard notation lengths.
- Added automatic merge of reducible tied segments during Normalize when no new onset needs to be shown.
- Changed app JSON loading so stored micro-rest segments are not reinserted directly; notes are restored first and rests are rebuilt by Normalize.
- Reduced mismatches between score notation immediately after load and after copy/paste round-trips.

Harmony / Accompaniment / Chords
- Added a song-level harmony reference track concept with save/load support.
- Unified accompaniment generation, AI accompaniment context, and top chord symbol display around a shared reference path.
- Harmony inference now prefers a temporary merged harmony reference built from all non-percussion tracks per measure.
- Excluded the target accompaniment track from the merged harmony source to avoid self-reference during accompaniment generation.

Editing / Interaction
- Added eraser tools.
  - Erase Measure: clears the clicked measure while keeping the measure slot.
  - Erase Note: deletes only the clicked note.
- Added `Adjust Velocity` in the Edit section.
  - Opens a curve-based velocity editor for the currently selected measure range on the selected track.
  - Shows note-position markers across the selected measures and lets the user raise or lower local velocity by dragging the curve.
  - Applies curve edits through undoable per-note velocity updates while preserving relative velocity differences inside chords.
- Simplified the top edit mode structure to two modes:
  - Measure Edit
  - Note Edit
- Measure selection is now available only in Measure Edit mode.
- Note selection is now available only in Note Edit mode.
- Removed right-click quick selection on the score.
- Removed long-press mouse pitch dragging for notes. Keyboard note movement remains available.
- When playback requires a selected measure range and no measure is selected, playback now falls back to the measure containing the last worked note.
- Playback no longer forces the UI to switch modes while playing.

Player / Palette / UI
- Added three playback start modes:
  - Selected range only
  - From selected range to end
  - Whole song
- Changed Play after Pause to resume from the current position instead of restarting from the beginning.
- Changed Ctrl+mouse-wheel zoom to follow the measure under the mouse cursor using score-space anchors, and kept the hovered measure's left boundary visible during reflow so zooming does not scroll past the measure start.
- Increased player control area and slider hit areas for easier volume and transport control.
- Increased the default startup window size and lowered the default score zoom so more measures are visible immediately after launch.
- Reorganized the left palette into these top-level sections:
  - File
  - Edit
  - Score Setup
  - Input
  - Track
  - Compose Assist
- Grouped Notes, Rests, Erasers, and Accidentals inside a single Input section, with rests reduced to a secondary row.
- Replaced the eraser icons with clearer large/small eraser visuals.
- Replaced single-track delete selection with a multi-select checklist popup so multiple tracks can be deleted at once.
- Hid the `Generate Lyrics Wave` command from the visible left palette/menu flow while keeping the underlying implementation intact.
- Changed the About dialog and app version API to read the executable file's `ProductVersion` from the Windows version resource, with fallback to the built-in project version string only when the resource cannot be queried.

Localization / Docs
- Added missing Korean localization keys in accompaniment-related menus that previously fell back to English.
- Changed startup localization loading to merge missing keys into existing language files, so older `ko.json` / `en.json` files pick up newly added menu strings automatically.
- Split runtime help content into dedicated language help files under `Language/HelpEng.json` and `Language/HelpKor.json`, and updated the help topics to reflect the current UI/workflow.
- Updated help text and developer documentation to reflect the new player modes, palette structure, eraser tools, and two-mode editing contract.

Notes

- Future changes should continue updating this file by adding or revising version sections in English.
